# Почтовый сервер alexol.io

Полноценное решение для корпоративной почты на собственном домене с веб-интерфейсом для администрирования и личных кабинетов пользователей.

## 🚀 Возможности

### Почтовый сервер
- ✉️ **SMTP сервер** для отправки и приема писем
- 📬 **IMAP сервер** для интеграции с почтовыми клиентами (Outlook, Thunderbird и др.)
- 🔐 **Авторизация пользователей**
- 📧 **Работа с письмами**: входящие, отправленные, удаление

### Админ-панель
- 👥 **Управление пользователями**: создание, удаление
- 📝 **Автоматическая генерация email** на основе логина (например: IIvanov@alexol.io)
- 📱 **Управление контактами**: ФИО, телефон, email

### Личный кабинет пользователя
- 📸 **Загрузка аватара** (хранится в MinIO)
- 🔑 **Изменение пароля**
- ✏️ **Редактирование профиля**: ФИО, телефон
- 📧 **Отправка и получение писем**

## 🔧 Настройка почтового клиента (Outlook)

### Настройки для Outlook

После создания пользователя в админ-панели, используйте следующие настройки:

#### Входящая почта (IMAP):
- **Сервер**: ваш-домен-или-ip
- **Порт**: 143
- **Безопасность**: Нет (для локальной сети) / STARTTLS (для продакшена)
- **Имя пользователя**: полный email (например: iivanov@alexol.io)
- **Пароль**: пароль пользователя

#### Исходящая почта (SMTP):
- **Сервер**: ваш-домен-или-ip
- **Порт**: 587 (или 25)
- **Безопасность**: Нет (для локальной сети) / STARTTLS (для продакшена)
- **Имя пользователя**: полный email
- **Пароль**: пароль пользователя

### Для продакшн-использования

⚠️ **ВАЖНО**: Для работы в продакшене необходимо:

1. **Настроить DNS записи**:
```
A     @           -> ваш-IP
A     mail        -> ваш-IP
MX    @           -> mail.alexol.io (приоритет 10)
TXT   @           -> "v=spf1 mx ~all"
```

2. **Настроить SSL/TLS**:
   - Получите SSL сертификат (Let's Encrypt)
   - Добавьте сертификаты в конфигурацию

3. **Изменить секретные ключи**:
   - В `docker-compose.yml` измените `SECRET_KEY`
   - Измените пароли для PostgreSQL и MinIO

4. **Настроить брандмауэр**:
   - Откройте порты: 25, 587, 143, 80, 443

## 🔌 API Endpoints

### Аутентификация
- `POST /api/auth/login` - Вход
- `GET /api/auth/me` - Текущий пользователь

### Администрирование
- `POST /api/admin/users` - Создать пользователя
- `GET /api/admin/users` - Список пользователей
- `DELETE /api/admin/users/{id}` - Удалить пользователя

### Профиль
- `PUT /api/profile` - Обновить профиль
- `POST /api/profile/avatar` - Загрузить аватар

### Почта
- `POST /api/emails/send` - Отправить письмо
- `GET /api/emails/inbox` - Входящие письма
- `GET /api/emails/sent` - Отправленные письма
- `GET /api/emails/{id}` - Получить письмо
- `DELETE /api/emails/{id}` - Удалить письмо
