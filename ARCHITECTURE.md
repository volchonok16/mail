## Компоненты системы

### 1. Frontend (React + TypeScript + Vite)
**Назначение**: Веб-интерфейс для пользователей и администраторов

**Технологии**:
- React 18 - UI библиотека
- TypeScript - типизация
- Vite - быстрая сборка
- TanStack Query - управление серверным состоянием
- Zustand - локальное состояние (auth)
- Nginx - веб-сервер

**Страницы**:
- `/login` - Авторизация
- `/admin` - Админ-панель (только для администраторов)
- `/dashboard` - Почтовый ящик пользователя
- `/profile` - Профиль пользователя

**Порты**: 3000 (dev), 80 (prod)

### 2. Backend (Python + FastAPI)
**Назначение**: REST API, SMTP/IMAP серверы, бизнес-логика

**Технологии**:
- FastAPI - веб-фреймворк
- SQLAlchemy - ORM
- Asyncpg - async PostgreSQL драйвер
- aiosmtpd - SMTP сервер
- Jose - JWT токены
- Passlib - хеширование паролей
- MinIO Client - работа с хранилищем

**Основные модули**:
- `main.py` - REST API endpoints
- `smtp_server.py` - SMTP сервер для приема/отправки
- `auth.py` - аутентификация и авторизация
- `models.py` - модели базы данных
- `minio_client.py` - загрузка файлов

**Порты**: 
- 8000 - REST API
- 25 - SMTP (прием писем)
- 587 - SMTP Submission (отправка)
- 143 - IMAP (чтение писем клиентами)

### 3. PostgreSQL
**Назначение**: Хранение данных

**Таблицы**:

#### users
```sql
- id: int (PK)
- email: string (unique)
- username: string (unique)
- full_name: string
- phone: string
- hashed_password: string
- avatar_url: string
- is_admin: boolean
- is_active: boolean
- created_at: timestamp
- updated_at: timestamp
```

#### emails
```sql
- id: int (PK)
- user_id: int (FK → users.id)
- from_address: string
- to_address: string
- subject: string
- body: text
- html_body: text
- is_read: boolean
- is_sent: boolean (true = отправленное, false = входящее)
- received_at: timestamp
```

**Порт**: 5432

### 4. MinIO
**Назначение**: Объектное хранилище для файлов

**Использование**:
- Хранение аватаров пользователей
- Bucket: `avatars`
- Публичный доступ на чтение

**Порты**:
- 9000 - API
- 9001 - Веб-консоль

## Потоки данных

### 1. Регистрация пользователя (Админ)
```
1. Админ открывает /admin
2. Заполняет форму (ФИО, логин, телефон, пароль)
3. POST /api/admin/users
4. Backend:
   - Генерирует email: {username}@alexol.io
   - Хеширует пароль (bcrypt)
   - Сохраняет в PostgreSQL
5. Возвращает созданного пользователя
```

### 2. Вход пользователя
```
1. Пользователь вводит email и пароль
2. POST /api/auth/login
3. Backend:
   - Проверяет email в БД
   - Проверяет пароль (bcrypt)
   - Генерирует JWT токен
4. Frontend сохраняет токен в localStorage
5. Все последующие запросы идут с заголовком:
   Authorization: Bearer {token}
```

### 3. Отправка письма (через веб)
```
1. Пользователь пишет письмо в /dashboard
2. POST /api/emails/send
3. Backend:
   - Сохраняет в БД (is_sent=true)
   - Отправляет через SMTP (localhost:25)
4. SMTP сервер получает письмо
5. Определяет получателя
6. Сохраняет в БД получателю (is_sent=false)
```

### 4. Получение письма (через IMAP)
```
1. Outlook подключается к IMAP (port 143)
2. Аутентификация (email + пароль)
3. Backend проверяет учетные данные
4. Outlook запрашивает список писем
5. Backend извлекает из PostgreSQL (is_sent=false)
6. Возвращает в формате IMAP
7. Outlook отображает письма
```

### 5. Загрузка аватара
```
1. Пользователь выбирает файл в /profile
2. POST /api/profile/avatar (multipart/form-data)
3. Backend:
   - Валидирует тип файла (image/*)
   - Генерирует уникальное имя
   - Загружает в MinIO (bucket: avatars)
4. MinIO возвращает URL
5. Backend сохраняет URL в users.avatar_url
6. Frontend обновляет аватар
```

## Безопасность

### Аутентификация
- JWT токены с подписью HS256
- Время жизни токена: 30 минут
- Пароли хешируются с bcrypt

### Авторизация
- Middleware проверяет токен в каждом запросе
- Разделение прав: admin / user
- Админ может управлять всеми пользователями
- Пользователь видит только свои письма

### Валидация
- Pydantic схемы для всех входных данных
- Email валидация
- Проверка типов файлов при загрузке

### CORS
- Настроен для фронтенда
- В продакшене ограничить origin

## Масштабирование

### Горизонтальное
- Backend можно запустить в нескольких экземплярах
- Нужен Load Balancer (nginx)
- PostgreSQL - один мастер, реплики для чтения
- MinIO поддерживает distributed mode

### Вертикальное
- Увеличить ресурсы контейнеров
- Настроить connection pooling для PostgreSQL
- Кеширование (Redis) для сессий и частых запросов

### Оптимизации
- Индексы в БД (email, user_id, received_at)
- CDN для статики фронтенда
- Сжатие gzip/brotli
- Lazy loading писем (пагинация)

## Мониторинг

Рекомендуемые инструменты:
- **Prometheus** - сбор метрик
- **Grafana** - визуализация
- **ELK Stack** - логирование

Метрики для отслеживания:

## Резервное копирование

### PostgreSQL
```bash
# Бэкап
docker-compose exec postgres pg_dump -U mailuser maildb > backup.sql

# Восстановление
docker-compose exec -T postgres psql -U mailuser maildb < backup.sql
```

### MinIO
```bash
# Бэкап bucket
mc mirror minio/avatars /backup/avatars

# Восстановление
mc mirror /backup/avatars minio/avatars
```
