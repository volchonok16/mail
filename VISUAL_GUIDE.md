## Поток работы с письмами

### Отправка письма:

```
1. Пользователь → Написать письмо
   ┌──────────────────────────┐
   │ Кому: user@example.com   │
   │ Тема: Hello              │
   │ Текст: Hi there!         │
   └──────────────────────────┘
         │
         ↓
2. Frontend → POST /api/emails/send
         │
         ↓
3. Backend → Сохраняет в PostgreSQL
         │     (is_sent = true)
         ↓
4. Backend → Отправляет через SMTP
         │
         ↓
5. SMTP Server → Получает письмо
         │
         ↓
6. Backend → Сохраняет получателю
              (is_sent = false)
         │
         ↓
7. Получатель → Видит во Входящих
```

### Получение письма (Outlook):

```
1. Outlook → Подключение к IMAP :143
         │
         ↓
2. Backend → Проверка логина/пароля
         │
         ↓
3. Outlook → Запрос списка писем
         │
         ↓
4. Backend → Извлекает из PostgreSQL
         │     (user_id = текущий, is_sent = false)
         ↓
5. Backend → Возвращает в формате IMAP
         │
         ↓
6. Outlook → Отображает письма
```

## Сервисы и их роли

```
┌─────────────────────────────────────────────────────┐
│  🐳 Docker Compose - Оркестрация                    │
└─────────────────────────────────────────────────────┘
   │
   ├─→ 🌐 Frontend Container (Nginx + React)
   │   ├─ Порт: 3000
   │   ├─ Задача: Веб-интерфейс
   │   └─ Проксирует /api → Backend
   │
   ├─→ ⚙️ Backend Container (FastAPI)
   │   ├─ Порт: 8000 (REST API)
   │   ├─ Порт: 25 (SMTP прием)
   │   ├─ Порт: 587 (SMTP отправка)
   │   ├─ Порт: 143 (IMAP)
   │   ├─ Задача: Бизнес-логика
   │   └─ Компоненты:
   │       ├─ REST API (FastAPI)
   │       ├─ SMTP Server (aiosmtpd)
   │       └─ Auth (JWT)
   │
   ├─→ 🗄️ PostgreSQL Container
   │   ├─ Порт: 5432
   │   ├─ Задача: Хранение данных
   │   └─ Таблицы: users, emails
   │
   └─→ 📦 MinIO Container
       ├─ Порт: 9000 (API)
       ├─ Порт: 9001 (Console)
       ├─ Задача: Хранение файлов
       └─ Bucket: avatars
```

## Безопасность

```
🔐 Многоуровневая защита:

┌─────────────────────────────────────────┐
│  1️⃣ Frontend                            │
│     └─ Проверка роутов (авторизация)   │
│                                         │
│  2️⃣ Backend API                         │
│     ├─ JWT токены (Bearer)             │
│     ├─ Middleware проверка токена      │
```
