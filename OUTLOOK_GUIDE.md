## 🚀 Быстрая настройка

### Шаг 1: Откройте Outlook

1. Запустите Microsoft Outlook
2. Перейдите в **Файл** → **Добавить учетную запись**
3. Выберите **Настроить вручную учетную запись**

### Шаг 2: Выберите тип учетной записи

- Выберите **POP или IMAP**
- Нажмите **Далее**

### Шаг 3: Настройте параметры

#### 👤 Сведения о пользователе:
- **Ваше имя**: Иванов Иван Иванович
- **Адрес электронной почты**: `IIvanov@alexol.io`

#### 📥 Входящая почта (IMAP):
- **Тип учетной записи**: IMAP
- **Сервер входящей почты**: `localhost` (или IP вашего сервера)
- **Порт**: `143`
- **Требуется SSL**: НЕТ (для локальной сети)

#### 📤 Исходящая почта (SMTP):
- **Сервер исходящей почты (SMTP)**: `localhost` (или IP вашего сервера)
- **Порт**: `587`
- **Требуется SSL**: НЕТ (для локальной сети)

#### 🔐 Вход в систему:

1. Нажмите **Проверить параметры учетной записи**
2. Outlook проверит подключение
3. Если все правильно - нажмите **Далее** → **Готово**

## ⚠️ Важные замечания

### Для локального использования:
- ✅ Используйте `localhost` или `127.0.0.1`
- ✅ Порты: IMAP 143, SMTP 587
- ✅ SSL/TLS: Отключено (не нужен для локальной сети)

### Для продакшн сервера:
- ⚠️ Настройте DNS записи для `mail.alexol.io`
- ⚠️ Получите SSL сертификаты (Let's Encrypt)
- ⚠️ Настройте порты с SSL: IMAP 993, SMTP 465/587
- ⚠️ См. [PRODUCTION.md](PRODUCTION.md)

## 🐛 Устранение неполадок

### Outlook не подключается

**Проблема:** "Невозможно установить соединение"

**Решение:**
1. Убедитесь, что backend запущен:
   ```bash
   docker-compose ps
   ```
2. Проверьте логи:
   ```bash
   docker-compose logs -f backend
   ```
3. Проверьте порты:
   ```bash
   telnet localhost 143
   telnet localhost 587
   ```

### Ошибка аутентификации

**Проблема:** "Неверное имя пользователя или пароль"

**Решение:**
1. Убедитесь, что используете **полный email** в качестве имени пользователя
2. Проверьте пароль (войдите через веб-интерфейс)
3. Попробуйте изменить пароль в профиле

### Письма не отправляются

**Проблема:** Письма в "Исходящих", но не отправляются

**Решение:**
1. Проверьте, что SMTP сервер запущен (логи backend)
2. Убедитесь, что порт 587 доступен
3. Попробуйте отправить письмо через веб-интерфейс
