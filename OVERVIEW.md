# 📧 Обзор почтового сервера alexol.io

## Что это?

Полнофункциональный почтовый сервер на собственном домене с:
- ✉️ Отправкой и приемом email
- 🖥️ Веб-интерфейсом (админка + личные кабинеты)
- 📱 Интеграцией с Outlook, Thunderbird и другими почтовыми клиентами
- 🔐 Безопасной аутентификацией
- 📸 Загрузкой аватаров
- 🎨 Современным UI

## Быстрый старт (Windows)

1. **Установите Docker Desktop**
   - Скачайте: https://www.docker.com/products/docker-desktop/
   - Запустите установку
   - Перезагрузите компьютер

2. **Запустите сервер**
   - Дважды кликните `start.bat`
   - Подождите 1-2 минуты

3. **Откройте браузер**
   - Перейдите на: http://localhost:3000
   - Войдите как админ:
     - Email: `admin@alexol.io`
     - Пароль: `Gord078134Alexol!9256`

4. **Создайте пользователя**
   - Нажмите "Создать пользователя"
   - Заполните форму (логин станет email)

5. **Отправьте письмо**
   - Войдите как созданный пользователь
   - Нажмите "Написать письмо"

**Готово! Ваш почтовый сервер работает! 🎉**

## Основные возможности

### Для администратора 👨‍💼

#### Создание пользователей
- Вводите ФИО, логин, телефон, пароль
- Email создается автоматически: `логин@alexol.io`

### Для пользователей 👤

#### Почта
- 📨 Отправка писем
- 📬 Получение писем
- 📂 Входящие / Отправленные
- 🗑️ Удаление писем

- 📸 Загрузка аватара
- 🔑 Смена пароля

- Настройте IMAP для получения
- Настройте SMTP для отправки
- Работает как обычная почта

## Сервисы и порты

| Сервис     | Порт  | Описание                          |
|------------|-------|-----------------------------------|
| Frontend   | 3000  | Веб-интерфейс (React)             |
| Backend    | 8000  | REST API                          |
| SMTP       | 25    | Прием писем                       |
| SMTP Sub   | 587   | Отправка писем                    |
| IMAP       | 143   | Чтение писем (Outlook)            |
| PostgreSQL | 5432  | База данных                       |
| MinIO API  | 9000  | Хранилище файлов                  |
| MinIO UI   | 9001  | Веб-консоль MinIO                 |

## Типичные сценарии использования

### Сценарий 1: Создание нового сотрудника
1. Админ входит в админ-панель
2. Создает пользователя: Иванов Иван → `IIvanov@alexol.io`
3. Отправляет учетные данные сотруднику
4. Сотрудник настраивает Outlook
5. Начинает работать с почтой

### Сценарий 2: Отправка письма через веб
1. Пользователь входит в личный кабинет
2. Нажимает "Написать письмо"
3. Заполняет: кому, тема, текст
4. Отправляет
5. Получатель видит письмо во "Входящих"

### Сценарий 3: Работа через Outlook
1. Пользователь настраивает Outlook (IMAP + SMTP)
2. Outlook подключается к серверу
3. Синхронизирует письма
4. Можно отправлять и получать как обычно

### Сценарий 4: Обновление профиля
1. Пользователь входит в "Профиль"
2. Загружает фото (сохраняется в MinIO)
3. Меняет пароль
4. Обновляет телефон
5. Изменения сохраняются

## Команды управления

### Запуск
```bash
# Windows
start.bat

# Linux/Mac
docker-compose up -d
```

### Остановка
```bash
# Windows
stop.bat

# Linux/Mac
docker-compose down
```

### Логи
```bash
docker-compose logs -f
```

### Перезапуск
```bash
docker-compose restart
```

### Полная очистка (удаляет все данные!)
```bash
docker-compose down -v
```

## Поддержка и помощь

### Проблемы при запуске?
1. Проверьте, что Docker Desktop запущен
2. Проверьте, что порты свободны (3000, 8000, 25, 587, 143)
3. Перезапустите Docker Desktop
4. Запустите заново: `start.bat`

### Не работает отправка писем?
1. Проверьте логи: `docker-compose logs backend`
2. Проверьте, что SMTP порт открыт
3. Попробуйте отправить через веб-интерфейс

### Не могу войти?
- По умолчанию админ: `admin@alexol.io` / `Gord078134Alexol!9256`
- Если изменили пароль и забыли - пересоздайте БД: `docker-compose down -v && docker-compose up -d`
